# Важно: команды nginx — НЕ в Windows!

## Что произошло

1. **SSH** — пароль `root` не подошёл (3 раза `Permission denied`).
2. **apt-get / nginx / certbot** вы ввели в **PowerShell на своём ПК** — там это не работает.  
   Эти команды только **на сервере Linux** после успешного входа `ssh root@83.222.18.141`.

---

## Шаг A. Узнать или сбросить пароль root

Сервер **83.222.18.141** — отдельный VPS. Пароль выдавал хостинг при создании сервера.

Где искать:

| Где | Что сделать |
|-----|-------------|
| Письмо при заказе VPS | Пароль root / «доступ к серверу» |
| Панель **Beget** → VPS / облако | Раздел сервера → **сброс пароля root** |
| Панель **Timeweb, Selectel, Reg.ru** и т.д. | То же, если VPS не Beget |
| Кто настраивал сервер за вас | Спросить пароль SSH |

В Beget часто есть: **Облако** → ваш сервер → **Консоль** / **VNC** / **Сброс пароля**.

Без правильного пароля (или SSH-ключа) на сервер не зайти — сначала только это.

---

## Шаг B. Войти на сервер

В PowerShell **только одна команда**:

```powershell
ssh root@83.222.18.141
```

Когда попросит `password:` — вставьте пароль **из панели** (в Windows при вводе символы не видны — это нормально).

Успех выглядит так — в начале строки будет что-то вроде:

```text
root@имя-сервера:~#
```

**Только после этого** вводите команды Linux.

---

## Шаг C. Одна команда на сервере (проще всего)

После входа по SSH (вы на сервере!):

```bash
cd /opt/aicrmpro/deploy && bash setup-nginx-on-vps.sh ваш@email.ru
```

Замените `ваш@email.ru` на реальный email (для SSL).

Если папки `/opt/aicrmpro` нет — сначала залейте проект (см. `deploy/upload-and-deploy.ps1`).

---

## Альтернатива: консоль в браузере (без SSH с ПК)

В панели хостинга VPS откройте **веб-консоль** / **VNC** — откроется терминал на сервере.  
Там те же команды из шага C.

---

## Проверка

В браузере на **вашем ПК**:

https://aicrmpro.ru/api/health

Должно быть: `{"status":"ok",...}`

---

## Частые ошибки

| Ошибка | Причина |
|--------|---------|
| `Permission denied` при ssh | Неверный пароль или вход только по ключу |
| `apt-get не найден` в PowerShell | Команды введены в Windows, а не на сервере |
| `certbot не распознано` в PowerShell | То же — certbot только на Linux-сервере |
