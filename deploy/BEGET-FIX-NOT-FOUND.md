# Ошибка «Not Found» на ссылке `/m/...` (aicrmpro.ru)

## В чём дело

Клиентская страница **уже работает** на сервере приложения, но домен **aicrmpro.ru** сейчас отдаёт **старый статический сайт** с хостинга Beget, а не ваш Docker на VPS.

| Проверка | Результат |
|----------|-----------|
| `http://83.222.18.141:3000/m/...` | ✅ 200, Express (приложение) |
| `https://aicrmpro.ru/m/...` | ❌ 404, nginx/Apache Beget |
| `https://aicrmpro.ru/` | Старая HTML-страница (не из Docker) |

DNS домена: **45.130.41.132** (Beget), VPS: **83.222.18.141**.

Пока домен не проксирует **все** запросы на порт **3000**, пути `/m/...`, `/login`, `/dashboard` будут давать 404.

---

## Решение 1 (рекомендуется): прокси с Beget на VPS

1. Войдите в панель **Beget** → сайт **aicrmpro.ru**.
2. Отключите или очистите **статический сайт** в `public_html` (старая заглушка), если он мешает.
3. Включите **обратный прокси / перенаправление на другой сервер** (название в панели может быть «Проксирование», «Nginx», «Node.js»):
   - **Цель:** `http://83.222.18.141:3000`
   - **Путь:** весь сайт `/` (не только главная).
4. Для webhook MAX (если нужен на том же домене):
   - `/webhook` → `http://83.222.18.141:3001`
5. SSL оставьте на Beget (как сейчас).

После сохранения подождите 1–5 минут и проверьте:

```text
https://aicrmpro.ru/api/health     → JSON {"status":"ok",...}
https://aicrmpro.ru/login            → страница входа (не 404)
https://aicrmpro.ru/m/ВАШ_КОД?ch=max → страница записи
```

В заголовках ответа должно быть `X-Powered-By: Express`, а не только `nginx-reuseport` без Express.

---

## Решение 2: DNS на VPS + nginx на сервере

1. В DNS (Beget или регистратор) задайте **A-запись** `@` и `www` → **83.222.18.141**.
2. На VPS установите nginx по примеру `deploy/nginx-aicrmpro.conf.example`.
3. Docker уже запущен:

```bash
cd /opt/aicrmpro
docker compose -f docker-compose.prod-external-ssl.yml up -d --build
```

4. Сертификат Let's Encrypt на VPS (certbot) или перенос существующих сертификатов.

---

## Решение 3: только статика на Beget (не рекомендуется)

Если заливаете только `frontend/dist` в `public_html` **без** Node:

- Скопируйте вместе с файлом **`.htaccess`** из `frontend/public/.htaccess` (попадает в `dist` при `npm run build`).
- API `/api` на Beget **не заработает** — нужен отдельный прокси `/api` → VPS:3000.

Для Wonder.ru правильный вариант — **прокси всего домена на :3000** (решение 1).

---

## Проверка с вашего ПК

```powershell
curl.exe -skI https://aicrmpro.ru/api/health
curl.exe -sI http://83.222.18.141:3000/api/health
```

Первый URL после исправления должен вести себя как второй (статус 200, Express).
