# Wonder.ru — что сделать в Beget (по вашим скринам)

## Главное одной фразой

**Приложение уже работает на сервере `83.222.18.141`.**  
Сейчас домен смотрит на Beget (`45.130.41.132`) — поэтому осьминог и ошибки.  
Нужно **переключить DNS на VPS** и **один раз настроить nginx на VPS**.

«Прилинковать домен к папке» на Beget **не нужно** — это для обычного сайта из файлов, не для Wonder.ru.

---

## Часть 1. Beget — DNS (ваш второй скрин)

1. Войти в **cp.beget.com** → **Управление DNS** → домен **aicrmpro.ru**.
2. Найти строку **A** для **aicrmpro.ru** (сейчас `45.130.41.132`).
3. Нажать **карандаш / изменить**.
4. В поле IP вписать: **`83.222.18.141`**
5. Сохранить.
6. То же для **www.aicrmpro.ru** (если есть A-запись) → тоже **`83.222.18.141`**.

Записи **MX** (почта) можно **не трогать**.

Подождать **15–60 минут** (иногда до 2 часов).

---

## Часть 2. VPS — nginx (один раз, по SSH)

С вашего компьютера (PowerShell), если есть пароль root:

```powershell
ssh root@83.222.18.141
```

На сервере выполнить **по очереди** (можно скопировать целиком):

```bash
apt-get update -qq
apt-get install -y nginx certbot python3-certbot-nginx

cat > /etc/nginx/sites-available/aicrmpro.ru << 'EOF'
server {
    listen 80;
    server_name aicrmpro.ru www.aicrmpro.ru;

    location /webhook {
        proxy_pass http://127.0.0.1:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
EOF

ln -sf /etc/nginx/sites-available/aicrmpro.ru /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
nginx -t && systemctl reload nginx

certbot --nginx -d aicrmpro.ru -d www.aicrmpro.ru --non-interactive --agree-tos -m ВАШ_EMAIL@mail.ru --redirect
```

Замените **`ВАШ_EMAIL@mail.ru`** на ваш реальный email (для Let's Encrypt).

Убедитесь, что Docker запущен:

```bash
cd /opt/aicrmpro
docker compose -f docker-compose.prod-external-ssl.yml up -d
curl -s http://127.0.0.1:3000/api/health
```

Должно быть: `{"status":"ok",...}`

## Часть 3. Проверка

В браузере:

1. **https://aicrmpro.ru/api/health** — текст `{"status":"ok"...}`
2. **https://aicrmpro.ru/login** — страница входа
3. Ссылка из кабинета **«Ссылки для клиентов»** — страница записи

---

## Про первый скрин (осьминог)

«Домен не прилинкован к директории» — это нормально, **пока DNS на Beget**.  
После смены A-записи на `83.222.18.141` сайт будет открываться с **VPS**, осьминог исчезнет.

**Не нажимайте** «прилинковать к public_html» ради Wonder.ru — без шагов 1–2 это не запустит CRM.

---

## Если нет доступа по SSH

Напишите хостингу / тому, кто давал сервер `83.222.18.141`:

> «Настройте nginx: aicrmpro.ru → localhost:3000, /webhook → :3001, SSL certbot. DNS уже на 83.222.18.141»

Или пришлите пароль/ключ SSH — можно настроить удалённо.
